ARE THE FEATURES WE SHIP ACTUALLY BEING USED?

This is a common question product managers all over the world ask themselves. Feature useage can be thought of as a signal for demand for that feature. Therefore, knowing feature useage can help product teams prioritize the next round of iterations on those set of features that are in high demand. As the user researcher with Wrap Media, I recently measured and analyzed feature useage by extracting event-data from a REST API and visualzing it for a weekkly feature useage dashboard. The outcome was a data-driven approach to prioritizing product sprints.

Approach - REST API and R

I set out to measure feature useage by accessing the relevant data through a rest API, and wrangle and visualize it using R.

At Wrap Media, we used FullStory which basically records and makes available to you any and all events that ever happen in your product. Fortunatley for us, they also have a neat REST API, so I had a wealth of data at my fingertips.

Together with the product team, we decided to measure weekly feature useage defined by the click events on that feature.

Creating the Data Set

Since the event data was accessible via the REST API, I decided to write a simple script to (a) identify the needed data, (b) extract the it, and (c) ceate a dataset that could then be wrangled and visualized.

1. Identify the data

Generating a LIST url with timestamp t-7 days:

t7 <- as.integer(Sys.time() - days(7))  
t7listURL <- paste(base_listURL, t7, sep= "")

Calling the API to get a list of data bundles for, the previous week:

list_response <- GET(t7listURL ,add_headers("Authorization" = paste("Basic", api_key)))

After some transformation from JSON to a table format, we succesfully identified the data bundles of the past week.

The next step was to pass the Ids to the a GET request to extract corresponding data bundles.

2. Exctracting the Data

I then wrote a simple for loop to extract each bundle, looping through the ids in Bundle Ids.

for (i in 1:length(bundle.ids)) {
  getURL <- paste(base_getURL, bundle.ids[i], sep="")
  name <- paste(bundle.ts[i], "bundle.gz", sep="-") 
  data <- GET(getURL,add_headers("Authorization" = paste("Basic", api.key)), write_memory()) %>%
    content(as="raw") %>%
    writeBin(con=name)
}

The individual bundles were then bound together into a dataframe for easy manipulation in R.
